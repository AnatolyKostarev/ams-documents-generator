# Краткий план DocGenerator MVP

## Суммарная оценка

| Показатель | Оценка |
|---|---:|
| Полная последовательная реализация (1 поток) | **130–160 ч** |
| Критический путь при параллели этапов 3 и 4 | **104–130 ч** |
| Ориентир по сроку (1 разработчик, full-time) | **16–20 раб. дней** |
| Ориентир по сроку (2 разработчика после этапа 2) | **13–16 раб. дней** |

## Этапы

| Этап | Что делаем | Оценка |
|---:|---|---:|
| 0 | Foundation монорепозитория (`pnpm`, `turbo`, общие пакеты) | 12–16 ч |
| 1 | Скелет `apps/web` (Next.js App Router + FSD + заготовки API) | 10–12 ч |
| 2 | Общий контракт (`packages/shared`) + Prisma + seed | 14–18 ч |
| 3 | Backend core в `apps/web` (`/api/generate`, `/api/pdf`, сессии) | 26–30 ч |
| 4 | Frontend core (маршруты, страницы, виджеты, UX-флоу) | 32–38 ч |
| 5 | Интеграция БД и same-origin API + обработка ошибок | 12–16 ч |
| 6 | SEO + rate limit + логи/алерты хостинга + Яндекс.Метрика (Sentry post-MVP) | 16–20 ч |
| 7 | Release Gate (pipeline, чеклист ТЗ, README, деплой) | 8–10 ч |

## Порядок выполнения

`0 -> 1 -> 2 -> (3 || 4) -> 5 -> 6 -> 7`

## Минимальные контрольные точки

- После этапа 2: контракт и Prisma полностью синхронизированы.
- После этапа 5: сквозной сценарий генерации и PDF работает.
- После этапа 7: выполнены критерии сдачи MVP по ТЗ.
